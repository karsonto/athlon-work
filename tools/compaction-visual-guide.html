Athlon 上下文压缩：静态 vs 动态 基于源码 ContextCompactionSettings 精确还原

静态模式：DynamicCompaction.Enabled = false — 在消息数 ≥ 50 或 Token ≥ 80,000（或窗口 70%）时 触发一次性 LLM 摘要，保留尾部 20 条消息。 动态模式（默认）：Enabled = true — 按预算利用率三级递进： ① 工具参数截断（≥72%）→ ② 前缀工具结果归档 → ③ LLM 对话摘要（≥80%），压缩后回落到 ~30% 窗口占用。
