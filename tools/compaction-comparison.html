Athlon 上下文压缩：静态 vs 动态（动画对比）

基于 ContextCompactionSettings 默认参数演示。 静态模式在固定消息数/Token 阈值触发「一次性 LLM 摘要 + 保留尾部」； 动态模式按预算利用率分三级（参数截断 → 工具结果归档 → LLM 摘要），压缩后回落到约 30% 窗口占用。

dynamicCompaction.enabled = false

三级递进时间线

正常累积，监控 TotalUtilization
≥72%（80%×0.9）→ ① 工具参数截断 truncate_args
≥72% 且压力升高 → ② 前缀工具结果归档 tool_result_eviction
≥80% → ③ LLM 对话摘要 conversation_compact
压缩后目标 ~30%（PostCompactionUtilization），并插入前端「上下文」卡片
用户 工具输出 参数截断 结果归档预览 LLM 摘要

参数来源：ContextCompactionSettings（TriggerTokens 80k / KeepMessages 20 / CompactTriggerRatio 0.7）与 DynamicCompactionSettings（TargetUtilization 80% / TruncateLeadRatio 90% / PostCompactionUtilization 30%）。 打开方式：双击本文件或在浏览器地址栏拖入 tools/compaction-comparison.html。
