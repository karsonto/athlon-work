# Windows 语音听写手动测试清单

在 Windows 10/11 实机上验证 Athlon 内置离线听写功能。

## 前置条件

1. 已安装「中文(简体)」或「英语(美国)」语音识别语言包（设置 → 时间和语言 → 语音）。
2. 系统已允许桌面应用访问麦克风（设置 → 隐私 → 麦克风）。
3. Athlon 设置页中「启用语音听写」已勾选，语言与系统语言包一致。

## 测试用例

1. **空输入框听写**：点击 Composer 麦克风 → 说「你好 Athlon」→ 输入框出现 interim/最终文本 → 再次点击结束 → 文本保留，未自动发送。
2. **追加听写**：输入框已有文字时开始听写 → 新内容追加在末尾，不覆盖原文。
3. **手动编辑中断**：听写过程中在输入框键入字符 → 听写自动停止，已识别文本保留。
4. **拒绝麦克风**：关闭麦克风权限 → 点击麦克风 → 显示明确错误提示，应用不崩溃。
5. **缺失语言包**：卸载当前识别语言包或设置不可用语言 → 麦克风按钮半透明禁用，ToolTip 提示安装语言包。
6. **会话切换**：听写中切换对话或关闭窗口 → 听写停止，无异常。
7. **发送流程**：听写结束后手动编辑 → Enter 发送 → 消息正常进入 Agent 流程。
8. **设置保存**：修改识别语言并保存 → 重新检测可用性，ToolTip/按钮状态更新。

## 已知限制

- 识别质量依赖 Windows 语言包与麦克风环境。
- 专业术语、代码符号容易识别错误，需用户手动修正。
- 部分语言包首次使用可能触发系统组件下载。
